# Security Policy

## Supported Versions

We are currently supporting the following versions of Byte School:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of Byte School seriously. If you believe you have found a security vulnerability, please report it to us as follows:

1. **Email us**: Send an email to security@byteschool.com with the details of the vulnerability.
2. **Do not open an issue**: Please do not report security vulnerabilities via public GitHub issues.

### What happens next?

- We will acknowledge receipt of your report within 48 hours.
- We will work with you to understand and reproduce the issue.
- Once fixed, we will coordinate a release and give you credit for the discovery (if you wish).

## Responsible Disclosure

We ask that you follow responsible disclosure principles:
- Give us a reasonable amount of time to fix the issue before making it public.
- Do not attempt to access or modify data that does not belong to you.
- Do not perform any actions that could degrade the service for other users (e.g., DoS).
